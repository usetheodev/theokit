# Deploy to Vercel

Manual deploy procedure for TheoKit apps. The build adapter produces a
Vercel-compatible artifact; this guide covers the operator side.

## Prerequisites

- Vercel CLI installed (`npm i -g vercel` or `pnpm dlx vercel`)
- Vercel account + project linked (`vercel link`)
- `STRIPE_WEBHOOK_SECRET`, `DATABASE_URL`, `CRON_SECRET`, and any other
  env vars set via `vercel env add`

## Build + deploy

```bash
# 1. Build for Vercel
pnpm theokit build --target=vercel

# 2. Deploy
vercel --prod
```

## What the build emits

- `.theo/crons.json` — neutral manifest of declared crons
- `.theo/jobs.json` — neutral manifest of declared jobs
- `vercel.json` — merged with your existing config (EC-105 preserves
  other fields). The `crons[]` array is replaced with TheoKit's
  declarations.
- `.vercel/output/` — Vercel Build Output API v3 artifacts (when the
  Vercel adapter ships full BOA emission)

## Cron secret

Vercel cron endpoints are publicly callable URLs. Protect them with the
`CRON_SECRET` env var pattern documented in
[Vercel Cron docs](https://vercel.com/docs/cron-jobs):

```ts
// In your cron handler — TheoKit's wrapper validates this automatically
// when the route stub is generated by the adapter translator.
```

## Cross-reference for multi-target deploys

If your `theo.config.ts` declares multiple `adapters`:

```ts
export default defineConfig({
  adapters: ['vercel', 'cloudflare'],
})
```

`theokit build --target=vercel` translates ONLY for vercel and emits a
stdout note suggesting the parallel `cloudflare` build. Per **EC-201**,
the `--target` flag is authoritative; `config.adapters[]` is informative.

To deploy to both:

```bash
pnpm theokit build --target=vercel
vercel --prod
pnpm theokit build --target=cloudflare
wrangler deploy
```

## Validation

After deploy, verify:

```bash
curl https://<your-app>.vercel.app/api/health     # 200 + JSON
curl https://<your-app>.vercel.app/api/__crons/<name> \
  -H "Authorization: Bearer $CRON_SECRET"         # fires cron handler
```

## SSE / streaming

Vercel supports SSE on Edge Functions. TheoKit's `defineAgentEndpoint`
emits SSE-compatible responses; the Vercel adapter wires them as Edge
Functions automatically.
