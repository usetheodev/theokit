## Issues

### 1. T6.1 WebSocket fixture wiring

**Severity:** MEDIUM

The Playwright spec for `tests/e2e/websocket-echo.spec.ts` runs but the
client cannot complete the WebSocket upgrade against the dev server.
Spec asserts `data-testid="ws-state"` reaches `'open'` — currently
times out at 5s. Investigation needed in
`fixtures/websocket-basic/node_modules` setup; the bare `theokit:`
symlink is not enough for the dev server to bootstrap the `ws`
transport for this fixture.

**Workaround:** WebSocket framework code IS covered by unit tests +
the existing `tests/unit/define-websocket.test.ts`. Spec is staged
and the wiring can be debugged in a follow-up PR.

### 2. T5.1 postgres + saas templates

**Severity:** LOW (blocked on infra, not code)

The 2 remaining template fixtures need a Postgres service in CI to
exercise their DB-backed flows. Pattern from `dashboard` + `api-only`
is locked — adding the 2 missing fixtures is a 1-2 day follow-up
after the CI workflow gets a Postgres service container + `pg_isready`
wait step (EC-9).

### 3. T4.1 + T9.1 real-world validation

**Severity:** LOW (blocked on secrets in this session)

The Vercel deploy + npm publish scripts are tested by their structural
suites + dry-run gates (atomic publish blocks partial publish via
EC-12). Real-world invocation needs `VERCEL_TOKEN` / `NPM_TOKEN` in
the CI release pipeline.
