## Confidence basis (why we believe CI will pass without manual iteration)

Local validation in T1.2 reproduced the EXACT same sequence:
- `psql CREATE DATABASE` × 2 ← same commands
- `pnpm exec drizzle-kit push --force --config <path>` × 2 ← same commands
- `npx playwright test --project=template-postgres` ← same command
- `npx playwright test --project=template-saas` ← same command

The only environmental delta CI → local:
- CI uses fresh `postgres:16-alpine` (postgres user/pass); local used existing `ankane/pgvector:v0.5.1` (themem user/pass)
- The pgvector container is a superset of postgres:16 functionality — no behavioral delta expected

Both Postgres containers expose 5432 and accept the standard psql + drizzle-kit clients. No reason CI would behave differently.
