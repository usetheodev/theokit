### Finding 3 — Drizzle migration tooling status (T0.2 motivation)

`drizzle-kit` was NOT installed anywhere in the repo at the start of this plan:

```
$ pnpm exec drizzle-kit --version
ERR_PNPM_RECURSIVE_EXEC_FIRST_FAIL  Command "drizzle-kit" not found
```

The fixtures `template-postgres` and `template-saas` have `drizzle.config.ts` files but neither `drizzle-kit` nor their parent monorepo had it as a dev dep. T0.2 fixes this.

### Finding 4 — Fixtures NOT in pnpm-workspace.yaml (T1.x command shape)

`pnpm-workspace.yaml` lists `fixtures/template-default`, `fixtures/theoui-autoinject`, `fixtures/services-*`. It does NOT list `fixtures/template-{dashboard,api-only,postgres,saas}`. Confirmed: `pnpm list -r` only shows the 5 workspace-registered fixtures.

**Consequence for the plan:** `pnpm --filter <fixture-name> exec ...` would fail with `No projects matched`. All `drizzle-kit push` invocations in T1.1 / T1.2 / T2.1 must use `pnpm exec drizzle-kit push --force --config <path>` from repo root (matches plan v1.1 ADR D2).

## Scope identified for Phase 1

The CI gap can be closed by adding a single new job `e2e-postgres-templates` in `.github/workflows/ci.yml` that:

1. Spins `postgres:16-alpine` service container
2. Sets `DATABASE_URL` (per database) and `THEO_SESSION_SECRET` (literal)
3. Creates 2 databases via `psql` (`CREATE DATABASE theokit_postgres_test`, `CREATE DATABASE theokit_saas_test`)
4. Runs `pnpm exec drizzle-kit push --force --config <fixture>/drizzle.config.ts` per fixture
5. Runs `npx playwright test --project=template-postgres` then `--project=template-saas` separately to control DATABASE_URL scope

## Decision

**PROCEED to T0.2 + Phase 1.** R0.5.3 is documented as done; no further code change for it. R0.5.2 closure requires only the new CI job + drizzle-kit install.
